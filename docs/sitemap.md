# サイトマップ

## 前提

- 最優先は「絵本として読む体験」。
- 現段階では、紙の絵本販売エリアは実装しない。
- 紙の絵本販売は将来フェーズとして設計には残す。
- Instagram から来たユーザーが、迷わず作品を読める構造にする。
- まずは No.010「蛍は森のナイトライト」を公開用サイトの基準作品として扱う。
- 作品一覧は、今後作品が増える前提で設計する。初期状態を 1 作品だけに最適化しすぎない。

## 設計で参照する基準

- [WCAG 2.2](https://www.w3.org/TR/WCAG22/) をアクセシビリティの基準として参照する。
- [Nielsen Norman Group の 10 Usability Heuristics](https://www.nngroup.com/articles/ten-usability-heuristics/) を、分かりやすさ、ユーザー操作の自由度、状態の見え方の判断基準として参照する。
- [Nielsen Norman Group の Mobile Navigation Patterns](https://www.nngroup.com/articles/mobile-navigation-patterns/) を、スマホで主要導線を絞る判断基準として参照する。
- [web.dev の INP ガイド](https://web.dev/articles/inp) を、操作反応の品質基準として参照する。
- [Google Search Central の helpful content ガイド](https://developers.google.com/search/docs/fundamentals/creating-helpful-content) を、検索と共有に耐えるページ内容の基準として参照する。

## ページ別の役割

### トップページ

目的:

- 初見ユーザーに世界観を伝える。
- すぐ絵本を読める導線を置く。
- Web サイトらしい説明を増やしすぎず、作品へ誘導する。

主な要素:

- ロゴ/サイト名
- 代表作品の大きなビジュアル
- 「読む」ボタン
- 「作品一覧」への導線
- Instagram への導線
- OGP/シェア時に伝わる短い説明文
- 主要画像の代替テキスト

現時点で入れない要素:

- 紙の絵本販売エリア
- 強い購入訴求
- 長いブランド説明

公開品質の要件:

- スマホのファーストビューで「何のサイトか」「何を読めるか」「どこを押せばよいか」が分かる。
- CTA は 1 つ目を「絵本を読む」に固定し、Instagram は副導線にする。
- 画像は重くなりやすいため、ファーストビュー画像は最適化し、不要な下層画像は遅延読み込みにする。
- キーボード操作でも主要導線へ移動できる。

### 作品一覧

目的:

- 作品が増えたときに並べられる場所を用意する。
- Instagram 投稿単位ではなく、絵本作品単位で整理する。

主な要素:

- 作品カード
- サムネイル
- 作品タイトル
- 短い紹介文
- 「読む」導線
- 公開状態
- 投稿日または公開日
- 将来的なカテゴリ/季節/テーマ

初期状態:

- No.010「蛍は森のナイトライト」を表示する。
- レイアウトは複数作品の追加を前提にしたカードグリッドにする。
- 作品数が少ない間も、ダミーの `Coming soon` カードで水増ししない。

公開品質の要件:

- 作品カード全体をタップ/クリックしやすくする。
- サムネイル、タイトル、短い紹介文だけで作品の違いが分かるようにする。
- 作品が増えたら、カテゴリ、季節、読了時間、公開日などで整理できる余地を残す。
- 初期から複数作品前提のデータ構造にする。

### 絵本ビュー

目的:

- 読む体験に集中させる。
- ナビゲーションや説明を最小限にする。

主な要素:

- 表紙
- 本文ページ
- 裏表紙
- ページ移動 UI
- ページ番号
- 画像の代替テキスト
- キーボード操作
- モーション軽減設定

既存方針:

- PC は見開き表示。
- スマホは画像の上に文章を重ねる。
- 作品タイトルは表紙のみ。
- ページ番号は紙面内に小さく表示する。

公開品質の要件:

- 左右スワイプだけに依存せず、タップ、クリック、キーボードでもページ移動できる。
- `prefers-reduced-motion` が有効な環境では、紙めくり演出を弱めるか無効化する。
- ページめくり中も操作反応が重くならないようにする。INP は 200ms 以下を目標にする。
- 文章は画像上に置く場合でも、読みやすいコントラストを確保する。
- 本文が画像としてしか存在しない状態にしない。HTML テキストとして保持する。

## SEO / OGP / SNS 共有

現時点で必要:

- トップページ、作品一覧、各作品ページに固有の `title` と `description` を用意する。
- 各作品ページに OGP 画像を設定する。
- Instagram から来たユーザーが共有しても、作品名と世界観が伝わる文言にする。
- 各作品ページは直接 URL で開ける構造にする。

将来検討:

- 構造化データ。
- 多言語展開。
- 紙本販売開始時の販売ページ OGP。

## 推奨する URL 設計

```text
/                 トップページ
/stories/         作品一覧
/stories/010/     No.010 絵本ビュー
/stories/{id}/    今後追加する作品の絵本ビュー
/books/           将来の紙本販売ページ
/about/           将来の紹介ページ
```

## 現時点の実装優先度

1. `/stories/010/` の絵本ビューを公開品質に近づける。
2. `/` のトップページを作る。
3. `/stories/` の作品一覧を作る。
4. データ構造を作品追加に耐える形へ整理する。
5. `/books/` はまだ実装しない。
