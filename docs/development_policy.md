# 開発方針

## 目的

AI 画像を使った絵本を Web サイトで公開し、その後に紙の絵本販売へつなげる。短期的には「絵本として読む体験」の品質を上げる。長期的には作品一覧、販売導線、紙本化を見据えた構成にする。

## 開発の優先順位

1. 絵本としてのページ構成を固める。
2. PC/スマホの読書体験を固める。
3. 作品一覧やトップページなど、サイト全体の導線を作る。
4. 公開品質として、アクセシビリティ、表示速度、SEO/OGP、SNS 共有を整える。
5. 販売導線、予約・購入・告知導線を設計する。
6. ページめくり、紙質感、音、細部アニメーションを本格的に作り込む。

## 現時点の進め方

- まずは大枠を優先する。
- ページめくりアニメーションは現状チープなため、品質フェーズで必ず再検討する。
- 仮実装では、過剰なリアル表現よりも、控えめで読書を邪魔しない動きを優先する。
- 作品一覧は、作品がすぐ増える前提で拡張しやすい構造にする。
- 作品追加の具体手順は `docs/story_addition_workflow.md` に従う。
- 管理番号は制作管理用であり、読者向けUIには原則表示しない。
- 重要なデザイン判断は `docs/design_decisions.md` に記録する。
- 新しい実装方針を決めたら、必要に応じてこのファイルへ反映する。

## フェーズ

### Phase 1: 絵本ビューの骨格

- 表紙、本文、裏表紙の構成を固める。
- 画像と本文の対応関係を固める。
- PC とスマホの基本読書体験を固める。
- 操作方法、ページ番号、タイトル表示ルールを固める。

### Phase 2: サイト構成

- トップページを設計する。
- 作品一覧を設計する。
- 絵本ビューへの導線を作る。
- Instagram から来たユーザーが迷わず読める構成にする。
- 各作品ページへ直接アクセスできる URL 設計にする。

### Phase 3: 公開品質の整備

- WCAG 2.2 を参照し、読めるコントラスト、キーボード操作、フォーカス表示、代替テキストを確認する。
- スマホでは主要 CTA、作品一覧、絵本ビューの操作を優先し、ナビゲーションを複雑にしない。
- ページめくりや画像演出は、`prefers-reduced-motion` に対応する。
- INP は 200ms 以下を目標にし、入力反応が重い演出を避ける。
- 各ページに固有の title、description、OGP を用意する。
- 大きな画像は最適化し、初期表示に不要な画像は遅延読み込みにする。

### Phase 4: 販売準備

- 紙の絵本販売を見据えたページを作る。
- 予約、購入、メール登録、SNS 導線を検討する。
- 価格、仕様、部数、販売方法は別途検討する。

### Phase 5: 体験品質の作り込み

- ページめくりアニメーションを本格的に作り込む。
- 紙の質感、ページ厚み、影、反り、光の変化を調整する。
- 必要に応じて Midjourney や画像生成で紙・表紙・背景素材を作る。
- 公開前に PC/スマホで視覚検証する。

## 保留事項

- リアルなページめくり演出の最終仕様。
- 紙本販売の具体的な導線。
- 作品数が増えた場合の一覧・カテゴリ設計。
- SEO、OGP、SNS シェア画像の具体文言。
- 本番公開先とビルド/デプロイ方法。

## 実装前の確認

Codex は、ユーザーが明確に実装を指示した場合を除き、実装前に方針を提示して了承を得る。調査、分析、案出しは実装ではないため、必要に応じて先に進めてよい。
