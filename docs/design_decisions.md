# デザイン決定事項

## 全体方針

- 最優先は「絵本として読む体験」。
- 日本国内向けに、大人が楽しめる静かで没入感のある絵本サイトを目指す。
- Web サイトらしい説明や装飾は控えめにし、作品そのものを主役にする。
- 見栄えだけでなく、読みやすさ、販売可能性、実装現実性を評価する。
- 公開品質では、アクセシビリティ、表示速度、SNS 共有、検索からの見え方も評価対象にする。

## 現在の採用案

- 作品体験は「ページをめくる絵本」形式を採用する。
- PC は見開き表示を基本にする。
- スマホは画像を上、本文とページ番号を下に分離し、1 ページずつ読む形式にする。
- スマホは左右スワイプと左右端タップでページ移動する。
- PC は本の左右に薄い矢印ボタンを置く。
- 作品一覧は複数作品が増える前提のカードグリッド構造にする。

## ページ比率

- 1 ページの比率は元 AI 画像と同じにする。
- 現在の基準画像は `896 x 1280`。
- 1 ページ比率は `7:10`。
- PC の見開き比率は `14:10`。
- スマホ本文ページでは、画像をトリミングせず元画像の比率を保って表示する。

## タイトル表示

- 作品タイトルは表紙のみに表示する。
- 本文ページにはタイトルを表示しない。
- 裏表紙にも大きなタイトルは表示しない。
- 読書中に外部 UI としてタイトルを重ねない。
- 表紙タイトルは必ず紙面内に収める。
- `titleLines` は最大 3 行までを目安にし、1 行は日本語で 7〜8 文字程度を基本にする。
- 短いタイトルでも大きくしすぎない。表紙タイトルは `clamp(24px, 3.2vw, 42px)` を基準にする。
- No.010 は `蛍は森の / ナイトライト` の 2 行で収める。
- No.011 は読者向けタイトルを `光の架け橋` とし、1 行表示を基準にする。
- タイトル領域は表紙内側の安全エリアに収め、横幅は紙面の 84% 程度を上限にする。
- `titleLines` の 1 要素は 1 行として扱い、行内では自動改行させない。
- タイトルが長い場合は、CSS の自動折り返しに頼らず、データ側の `titleLines` で自然な位置に改行を入れる。
- 表紙にはタイトル以外のサブ文章を置かない。作品データにも `cover.text` は持たせない。

## ページ番号

- PC では右側の文章ページ右下にページ数を小さく表示する。
- スマホでは本文エリア右下にページ数を小さく表示する。
- ページ番号は読書体験を邪魔しない濃度・サイズにする。
- 外部トーストとしてページ番号・タイトルを表示しない。

## 操作 UI

- PC の左右ボタンは本の外側に薄く表示する。
- スマホではボタンを表示せず、左右スワイプと左右端タップで操作する。
- 中央タップでページ情報を出す仕様は廃止した。
- キーボードの左右キーでもページ移動できるようにする。
- フォーカス表示は消さない。絵本体験を壊さない範囲で見えるように調整する。

## アニメーション

- 生成画像で作った表紙、裏表紙、紙テクスチャを使用し、タイトルや本文は Web 側で重ねる。
- 紙テクスチャは本文の可読性を邪魔しない濃度で薄く重ねる。
- ページ端の厚み、綴じ目、めくり中の影は CSS で表現する。
- ページめくりアニメーションは v2.2 とし、現在ページと次ページの HTML を一時複製して軽いフェード、浅い回転、横抜けで動かす。
- 紙の厚みやリアルな紙のしなりよりも、軽くふわっとめくれる快適さを優先する。
- PC もスマホも回転角と影をかなり抑え、ページ端の主張を弱くする。
- リアルな物理シミュレーションや WebGL 化までは行わない。
- `prefers-reduced-motion` が有効な場合は、ページめくり演出を弱めるか無効化する。
- アニメーションは読書体験の補助であり、本文の読みやすさや操作反応より優先しない。

## 作品一覧

- 作品が増える前提で、カード追加に耐える設計にする。
- 初期作品が少ない場合でも、空の `Coming soon` カードで水増ししない。
- カードにはサムネイル、タイトル、短い紹介文、テーマタグを持たせる。
- 将来的に、季節、テーマ、公開日などで整理できる余地を残す。

## 画像とパフォーマンス

- 画像は作品体験の中心なので、大きく見せる。
- 表紙・裏表紙は作品ごとに差し替える。共通素材はフォールバックとして残す。
- 表紙・裏表紙の生成画像には登場人物を描かない。作品世界を示す場所、道具、光、余韻で構成する。
- ただし、初期表示に不要な画像は遅延読み込みにする。
- ファーストビューの画像は優先読み込みを検討する。
- 入力反応の品質として、INP 200ms 以下を目標にする。

## SEO / OGP

- トップページ、作品一覧、各作品ページに固有の title と description を用意する。
- 各作品ページに OGP 画像を設定する。
- Instagram から共有されたときに、作品名と世界観が伝わる文言にする。

## 別バージョンの扱い

- このリポジトリでは、ページめくり絵本版だけを管理する。
- 過去に検討した没入スクロール版や静的見開き版は、制作管理側の親フォルダで保管する。

## 読者ペルソナと本文方針

- 読者ターゲットは、3〜6歳の子どもと、その子どもに読み聞かせる親・祖父母を中心にする。
- 購入判断者は主に大人であるため、子どもが楽しめることに加えて、親が安心して読ませられる内容にする。
- 基本体験は「寝る前に親子で読む、やさしい森の絵本」とする。
- 夜の話だけに限定しない。春夏秋冬、朝・昼・夕方・夜、晴れ・雨など、季節感と時間帯の幅を持たせる。
- 季節カテゴリは春・夏・秋・冬を基本にし、梅雨など日本の季節感も作品整理の軸にする。
- 本文は、声に出して読みやすい短い文を基本にする。
- 1文は短くし、3〜6歳が場面を追える言葉を使う。
- 難しい比喩、抽象的すぎる余韻、大人向けの詩的表現に寄せすぎない。
- 怖さ、寂しさ、不安を強くしすぎない。
- 各話の最後は、安心、やさしさ、満足感、眠りや次の日への期待につながる終わり方にする。
- 繰り返しの言葉や音のリズムを適度に使い、読み聞かせで気持ちよく読める文章にする。
- 説教や教訓を前面に出さず、行動や情景から自然に伝わるようにする。
- かわいらしさは保つが、幼すぎる言葉だけに寄せず、紙の絵本として手元に置きたくなる品のよさも残す。
