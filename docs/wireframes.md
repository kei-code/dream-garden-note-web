# ワイヤーフレーム

## 前提

- このワイヤーフレームは公開用サイトの大枠を決めるためのもの。
- 紙の絵本販売エリアは現段階では画面実装しない。
- 販売導線は将来追加できるよう、設計メモとして残す。
- ページめくりのリアル演出は後の品質フェーズで作り込む。
- 作品一覧は今後作品が増える前提で設計し、1 作品だけの特設ページのようにはしない。
- アクセシビリティ、スマホ操作、表示速度、SNS 共有まで含めて公開品質を判断する。

## 共通設計ルール

- 主要 CTA は画面内で迷わない位置に置く。
- クリック/タップできる要素は、見た目だけでなく操作対象として十分な大きさにする。
- キーボード操作時はフォーカス位置が分かるようにする。
- 画像だけで意味が完結しないよう、作品名、本文、説明は HTML テキストとして持つ。
- 大きな画像は初期表示に必要なものを優先し、それ以外は遅延読み込みにする。
- 強いアニメーションは `prefers-reduced-motion` に対応し、モーション軽減時は演出を弱める。

## 1. トップページ

### PC

```text
┌──────────────────────────────────────────────────────────────┐
│ DGN / Dream Garden Note                  Stories Instagram  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│             [ 大きな作品ビジュアル / 森の夜の絵 ]              │
│                                                              │
│                 蛍は森のナイトライト                         │
│           静かな夜の森をめくる、小さなWeb絵本                 │
│                                                              │
│                 [ 絵本を読む ]   [ 作品一覧 ]                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

方針:

- ファーストビューで作品を見せる。
- 説明は短くする。
- メイン CTA は「絵本を読む」。
- 販売導線はまだ出さない。
- ヘッダーは薄くし、作品ビジュアルの邪魔をしない。
- 「作品一覧」は将来の作品追加を見据えて常設する。
- Instagram は副導線に留める。

- ファーストビュー内に「絵本を読む」ボタンを入れる。
- 画像上に文字を重ねる場合は、文字の背面に控えめな暗いレイヤーを置く。
- ヘッダーのリンク数は絞る。主要導線は「読む」「作品一覧」。

方針:

- 作品が増えたら、季節、テーマ、公開日で絞り込む余地を残す。

## 3. 絵本ビュー

### PC

```text
┌──────────────────────────────────────────────────────────────┐
│ DGN                                           Top / Stories   │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   [‹]        ┌──────────────┬──────────────┐        [›]      │
│              │              │              │                 │
│              │  画像ページ  │  文章ページ  │                 │
│              │              │              │                 │
│              │              │        2 / 6 │                 │
│              └──────────────┴──────────────┘                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

方針:

- PC は見開きを最大限大きくする。
- タイトルは表紙のみ。
- 文章ページ右下にページ数を置く。
- 左右ボタンは本の外側に薄く置く。
- 本の上にページ情報を重ねない。
- 左右ボタンには分かるラベルを付け、キーボードフォーカス時は見えるようにする。
- キーボードの左右キーでもページ移動できるようにする。
- ページめくり中に連打しても破綻しないよう、入力制御を入れる。
- モーション軽減時は紙めくりではなく、短いフェードまたは即時切り替えにする。

### スマホ

```text
┌──────────────────────┐
│ DGN                  │
├──────────────────────┤
│                      │
│ ┌──────────────────┐ │
│ │                  │ │
│ │    画像全面       │ │
│ │                  │ │
│ │                  │ │
│ │  本文テキスト     │ │
│ │             2 / 6│ │
│ └──────────────────┘ │
│                      │
│ 操作: 左右スワイプ   │
│      左右端タップ    │
└──────────────────────┘
```

方針:

- スマホはボタンを表示しない。
- 画像の上に本文を重ねる。
- ページ数は画像ページ右下に小さく表示する。
- 左右スワイプと左右端タップで操作する。
- 端タップ領域は視覚的には目立たせず、操作対象としては十分な幅を確保する。
- スワイプだけに依存しない。左右端タップも必ず維持する。
- ブラウザ戻るジェスチャーとの干渉を抑える。
- 本文は背景に暗いグラデーションを敷き、読みやすさを優先する。

## 4. 状態設計

### 読み込み中

```text
┌──────────────────────┐
│ DGN                  │
├──────────────────────┤
│                      │
│ [ 低解像度プレースホルダー ] │
│                      │
│ 読み込み中           │
│                      │
└──────────────────────┘
```

方針:

- 白紙のまま待たせない。
- 初期表示に必要な表紙画像を優先して読み込む。
- 以降のページ画像は遅延読み込みにする。

### エラー

```text
┌──────────────────────┐
│ DGN                  │
├──────────────────────┤
│ 画像を読み込めません │
│ [ 再読み込み ]       │
│ [ 作品一覧へ戻る ]   │
└──────────────────────┘
```

方針:

- 何が起きたかを短く伝える。
- 再読み込みと作品一覧への退避導線を置く。
- 画像が失敗しても本文テキストは読める構造にする。
